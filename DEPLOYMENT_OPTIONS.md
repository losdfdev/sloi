# Инструкция по деплою (Render)

Приложение (Frontend + Backend) полностью подготовлено для простого деплоя на **Render.com** как единый сервис. Бэкенд теперь умеет автоматически раздавать собранный фронтенд.

## Шаг 1. Деплой "в один клик" на Render (Web Service)
Это самый удобный способ получить рабочее приложение с HTTPS (`https://your-app-name.onrender.com`).

1. **GitHub**: Запушьте весь проект в репозиторий на GitHub.
2. **Render Settings**: Зарегистрируйтесь на [Render.com](https://render.com), нажмите **New +** -> **Web Service** и выберите ваш репозиторий.
3. Укажите следующие настройки:
   - **Name**: Название вашего проекта (например, `sloi-app`)
   - **Root Directory**: `backend` (ВАЖНО! Указываем именно эту папку)
   - **Environment**: `Node`
   - **Build Command**: `npm install && npm run build` (эта команда установит зависимости бэкенда, а затем автоматически установит зависимости и соберет папку `frontend`).
   - **Start Command**: `npm start`
4. **Environment Variables**: Нажмите "Advanced" и добавьте переменные:
   - `TELEGRAM_BOT_TOKEN` = Ваш токен бота
   - `SUPABASE_URL` = Ссылка на ваш проект Supabase
   - `SUPABASE_ANON_KEY` = Ваш ключ Supabase
   - `NODE_ENV` = `production` (КРИТИЧЕСКИ ВАЖНО: это скажет бэкенду, что нужно работать в режиме раздачи статики браузеру).

5. Нажмите **Create Web Service**. В течение 3-5 минут Render соберет приложение и запустит его.

## Шаг 2. Подключение Telegram бота
После того как сервис будет "Live", Render выдаст вам ссылку вида `https://sloi-app.onrender.com`.

1. Скопируйте эту ссылку.
2. Перейдите в Telegram к **@BotFather**.
3. Настройте кнопку Web App (`/setmenubutton` -> выберите бота -> отправьте ссылку).
4. Настройте стартовую команду (если используете бота как лончер), чтобы он отправлял кнопку Mini App c этой ссылкой.

Готово! Теперь ваше приложение доступно миру и работает 24/7.

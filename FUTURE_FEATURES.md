# Sloi App - Будущие Фичи и План Развития

Проект сейчас находится в стабильной стадии MVP (Минимально Жизнеспособный Продукт). Фронтенд успешно развернут на Render, базовые механики (онбординг, премиум, лимиты свайпов, профили) полностью функционируют. 

## Следующие шаги (Phase 2): Чат и Социальное Взаимодействие
Мэтчинг уже работает, но пользователи не могут общаться. Это следующая важнейшая часть.

1. **База данных для сообщений:**
   - Создать таблицу `messages` (id, sender_id, receiver_id, match_id, content, created_at, is_read).

2. **Бэкенд-логика (WebSockets):**
   - Интегрировать библиотеку `socket.io` в `backend/src/index.js`.
   - Настроить безопасную передачу токенов по веб-сокетам, чтобы видеть, кто "онлайн" и мгновенно отправлять сообщения в обход стандартных HTTP запросов.
   - Использовать уже созданный нами флаг `hide_online_status`, чтобы скрывать юзера из списка "в сети".

3. **Фронтенд Чатов:**
   - Создать полноценную страницу со списком диалогов (мэтчей).
   - Создать интерфейс самого чата (пузырьки сообщений, поле ввода).
   - Подключить клиентский `socket.io-client` для реактивного обновления.

## Следующие шаги (Phase 3): Админка и Модерация
Так как приложение работает со случайными юзерами, потребуется базовый контроль:
1. **Роли пользователей:** Добавить `role` (user/admin) в таблицу Supabase.
2. **Панель Администратора:** Отдельный скрытый роут во фронтенде с дашбордом:
   - Просмотр всех пользователей и их жалоб.
   - Кнопка "Забанить" (Banned status).
3. **Система репортов:** Возможность нажать на "Пожаловаться" прямо в карточке `SwipeCard`, если там неприемлемый контент.

## Монетизация и Расширение
- **Telegram Stars:** Мы внедрили имитацию проверки оплаты, в будущем можно будет привязать настоящего Telegram Payments Bot для автоматической сверки чеков.
- **Push-уведомления (Telegram Bot):** Бот уже настроен отправлять уведомления о лайках и мэтчах. Можно добавить рассылки администратора (например, "Обновление функционала! Зайди посмотреть").
